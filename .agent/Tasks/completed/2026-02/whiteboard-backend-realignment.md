# Whiteboard Backend Realignment — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Realign the TurfSheet database schema and TypeScript types to match the actual whiteboard workflow — one primary job per staff, free-text second jobs list with grab-to-complete semantics, priority letters, daily metadata, carryover logic, and skill-tracking for future smart suggestions.

**Architecture:** The current schema has two structural problems: (1) `daily_assignments` supports `job_order` 1 or 2, but the real workflow is one primary job per person, and (2) `second_job_board` references `jobs.id` via foreign key, but second jobs are ad-hoc free text, not template references. This plan restructures those tables, adds daily board metadata (sunrise, announcements), adds a `staff_skills` table for learning, and updates TypeScript types. Frontend changes are explicitly OUT OF SCOPE — this is backend only.

**Tech Stack:** PostgreSQL (Supabase), TypeScript (types only), Supabase CLI for migrations

**Current Schema Location:** All tables in `turfsheet` schema. Migrations in `supabase/migrations/`.

**Reference Photo:** Physical whiteboard showing staff list (left), one primary job per person (center), free-text daily task list (right), sunrise time + announcements (top right).

## Context for the Implementing Engineer

### The Physical Whiteboard Workflow

The superintendent runs a physical whiteboard each morning at a golf course maintenance shop:

1. **Left side:** Staff names in a column. One job written next to each name (mow, roll, irrigate, etc.).
2. **Right side:** A free-text list of "second jobs" — ad-hoc tasks like "Blow decks + mats", "Branches (3/17)", "Fence post (12)". These are shorthand, situational, NOT from a template library.
3. **Top right:** Day of week, sunrise time, announcements (e.g., "B-9 START" = back nine start).

**After primary jobs are done**, staff return and the super has prioritized the second jobs list with **letters** (A–Z, where A = emergency). The letters are NOT sequential — they reflect urgency. The super assigns them all at once. Staff either get assigned by the super or self-select ("grab") from the list.

**Grabbing = completing.** When a staff member grabs a second job, it's crossed off. No separate "mark done" step. Uncompleted jobs carry forward to the next day automatically.

**Primary jobs are template-based** and will eventually be auto-generated by a rules engine (mow greens every day, bunkers every day, fairways alternating groups, etc.). Second jobs are free text and ad-hoc.

### What's Wrong with the Current Schema

| Problem | Current State | Required State |
|---------|--------------|----------------|
| Primary assignments | `job_order IN (1, 2)` — two slots per staff | One slot per staff per date. Remove `job_order`. |
| Second jobs data source | `second_job_board.job_id → jobs.id` (FK to template) | Free text field `description`. No FK to jobs table. |
| Second jobs completion | `status: 'pending' \| 'assigned'` + separate `second_job_assignments` table | Grabbing = completing. Single `grabbed_by` + `grabbed_at` on the board item. |
| Priority letters | Not on second jobs board | `priority CHAR(1)` on `second_job_board` |
| Daily metadata | Not modeled | New `daily_board` table for sunrise, announcements, date |
| Staff skills | Not modeled | New `staff_skills` table tracking what staff have done |
| Carryover | Not modeled | Query-based: uncompleted items from yesterday get copied forward |

### Database Schema (turfsheet schema)

**Existing tables being modified:**
- `turfsheet.daily_assignments` — remove `job_order`, add unique on `(staff_id, assignment_date)`
- `turfsheet.second_job_board` — replace `job_id` FK with `description` text, replace `status`/`rank` with `priority`/`sort_order`/`grabbed_by`/`grabbed_at`

**Existing table being dropped:**
- `turfsheet.second_job_assignments` — no longer needed (grab is on the board item itself)

**New tables:**
- `turfsheet.daily_board` — daily metadata (date, sunrise, announcements)
- `turfsheet.staff_skills` — tracks staff capabilities (hardcoded + learned)

**Existing tables unchanged:**
- `turfsheet.jobs` — still used for primary job templates
- `turfsheet.staff` — unchanged

## Post-Implementation: Schema Summary

After all tasks complete, the `turfsheet` schema should contain:

```
turfsheet.staff (unchanged)
├── id, role, name, telephone, telegram_id, notes

turfsheet.jobs (unchanged)
├── id, title, description, crew_needed, priority, section

turfsheet.daily_board (NEW)
├── id, board_date (UNIQUE), sunrise_time, announcements

turfsheet.daily_assignments (MODIFIED)
├── id, staff_id → staff, job_id → jobs
├── assignment_date
├── notes, completed_at
├── UNIQUE(staff_id, assignment_date) — one primary job per person per day
├── REMOVED: job_order, priority

turfsheet.second_job_board (RESTRUCTURED)
├── id, board_date, description (free text)
├── sort_order, priority (A-Z letter)
├── grabbed_by → staff, grabbed_at
├── carried_from (date, for carryover tracking)
├── REMOVED: job_id FK, status, rank
├── DROPPED: second_job_assignments table entirely

turfsheet.staff_skills (NEW)
├── id, staff_id → staff, skill_name
├── source (hardcoded | learned), times_completed, last_completed_at
├── UNIQUE(staff_id, skill_name)
```
